## 简介
本文件面向产品经理、业务分析师与开发者，系统性梳理平台的商城业务模块，覆盖商品管理、订单处理、用户体系、营销活动等核心业务域。文档基于实际代码与数据库结构，提供业务流程、数据模型与接口设计的参考，帮助快速理解与落地实现。

## 项目结构
平台采用多模块分层架构：
- platform-admin：后台管理接口服务
- platform-api：商城/小程序 API 服务
- platform-biz：核心业务逻辑、Service、DAO、MyBatis Mapper
- platform-common：公共配置、工具、缓存与通用能力
- 平台前端：platform-admin-ui（Vue2 + ElementUI）、wx-mall（原生小程序）、uni-mall（uni-app）

## 架构总览
系统采用“前端小程序/管理端 + API 层 + 业务层 + 数据持久化”的分层设计。支付通过微信支付通道完成，后台管理端负责订单发货与运营配置。

```mermaid
sequenceDiagram
participant U as "用户"
participant C as "小程序客户端(uni-mall/wx-mall)"
participant API as "platform-api"
participant BIZ as "platform-biz"
participant R as "Redis"
participant DB as "MySQL"
participant PAY as "微信支付"
participant AUI as "后台管理端(platform-admin-ui)"
participant ADM as "platform-admin"
%% 1) 注册/登录
U->>C : "微信授权登录/注册"
C->>API : "/app/loginByWeixin"
API->>BIZ : "用户注册或登录逻辑"
BIZ->>DB : "新增/更新用户"
DB-->>BIZ : "用户记录"
BIZ-->>API : "token + userInfo"
API-->>C : "登录成功"
C->>R : "缓存会话(token)"
%% 2) 下单
U->>C : "选择商品并提交订单"
C->>API : "/app/cart/checkout"
API->>BIZ : "计算价格/优惠券/运费"
BIZ->>DB : "查询购物车/商品/优惠券/地址"
DB-->>BIZ : "数据结果"
BIZ-->>API : "结算信息"
API-->>C : "结算页数据"
U->>C : "确认下单"
C->>API : "/app/order/submit"
API->>BIZ : "创建订单"
BIZ->>DB : "写入订单主表+订单明细"
DB-->>BIZ : "订单ID/编号"
BIZ-->>API : "orderId/orderSn"
API-->>C : "下单成功"
%% 3) 支付
U->>C : "发起支付"
C->>API : "/app/pay/prepay(orderId)"
API->>PAY : "统一下单/预支付"
PAY-->>API : "prepay参数"
API-->>C : "返回支付参数"
C->>PAY : "调起微信支付SDK"
PAY-->>API : "支付回调(notify)"
API->>BIZ : "更新支付状态"
BIZ->>DB : "订单状态=已支付(待发货)"
%% 4) 发货
AUI->>ADM : "后台发货(订单ID+物流单号)"
ADM->>BIZ : "发货业务校验"
BIZ->>DB : "更新发货状态/物流信息"
DB-->>BIZ : "更新成功"
BIZ-->>ADM : "发货成功"
ADM-->>AUI : "状态变更(待收货)"
%% 5) 确认收货
U->>C : "确认收货"
C->>API : "/app/order/confirmOrder"
API->>BIZ : "收货确认"
BIZ->>DB : "订单状态=已完成"
DB-->>BIZ : "更新成功"
BIZ-->>API : "完成"
API-->>C : "确认收货成功"
```

## 性能考虑
- 缓存策略：利用 Redis 缓存会话、热点数据与接口结果，降低数据库压力
- 分页与索引：订单、商品、优惠券等高频查询需合理分页与索引设计
- 异步处理：支付回调、发货通知等可异步处理，避免阻塞主流程
- 读写分离：高并发场景建议引入读写分离与连接池优化

## 故障排查指南
- 登录/会话异常：检查 Redis 会话是否正常、token 生成与校验逻辑
- 下单失败：核对购物车、库存、价格计算与优惠券可用性
- 支付回调：确认回调地址、签名验证与幂等处理
- 发货异常：核对订单状态流转、物流信息与后台校验逻辑
- 数据一致性：关注事务边界与分布式锁使用
